JavaScript Objects

  • Almost everything is an object in JavaScript
  • Booleans, Numbers, and Strings if defined with new keyword
  • Dates, Maths, Regular expressions, arrays, functions, and objects are always objects
  • A good example is a car is an object and it has properties like weight and color. And methods like start and stop.

Inheritance

  • Uses Prototypal inheritance

Classical uses copies of properties behaviors. They are seperate entities. v1, v2 are instances.

v1, v2 are link to Vehicle.prototype using keyword new. No copies, but links. These are ptotype chain.


function testRun() {
//Vehicle - superclass function
Vehicle(name) { this.name = name; }
// superclass method Vehicle.prototype.start = function () {
return "engine of " + this.name + " starting..."; };
// Car - subclass
function Car(name) {
Vehicle.call(this, name);
// call super constructor.
}
// subclass extends superclass Car.prototype = Object.create(Vehicle.prototype);
// subclass method Car.prototype.run = function () {
console.log("Hello " + this.start()); };
// instances of subclass
var c1 = new Car("Fiesta"); var c2 = new Car("Baleno");
// accessing the subclass method which internally access superclass method
c1.run();
// "Hello engine of Fiesta starting..."
c2.run();
// "Hello engine of Baleno starting..." }

Instatiation

Ways to create things in JavaScript

Four instatiation patters:

  • Functional
  • Functional-shared
  • Protoypal
  • Psedoclassical

Functional

  • Create function
  • Create empty object
  • Add properties and methods

Functional-shared

  • Overcomes having to make a duplicate of the method in memery every time you create new object.
  • Methods are defined in another object and then we extend our object with these methods.

Prototypal

  • Uses prototype chain to create objects
  • Methods are attached to object's prototype using Object.create method

Psedoclassical

  • Uses prototype chain to create objects
  • Instead of using Object.create, it uses "this"
